# Standing Restrictions

BEFORE taking any of the following actions, you MUST ask for explicit approval and wait for confirmation. These restrictions override all other instructions. No exceptions.

| Action | Rule |
|--------|------|
| Create a new Blueprint | Ask first — always |
| Overwrite an existing Blueprint | Ask first — always |
| Build new Equipment | Only when a genuine gap exists — ask first |
| Re-run after API failure | Confirm before re-running — "This will make an API call. Proceed?" |
| Delete any file | Confirm — irreversible. Prefer moving to archive/ |
| Send external messages | Confirm before sending — show the draft first |
| Consequential decisions | Describe the decision and options — never choose unilaterally |
| Publish anything publicly | Show the content and confirm before posting |
| Modify .env or credentials | Confirm before touching |
| Send client emails | Always save as draft — never send directly. Abderrahim reviews all outbound client emails before they go. |
